## 1. Introducción a Supabase

### ¿Qué es Supabase?

**Supabase** es una alternativa open-source a Firebase que proporciona:

- **Base de datos PostgreSQL** completa
- **Autenticación** integrada
- **Storage** para archivos
- **Realtime** subscriptions
- **Edge Functions** (serverless)
- **API REST automática** generada desde tu esquema

```
┌─────────────────────────────────────────────────────────────┐
│                         Supabase                             │
├─────────────┬─────────────┬─────────────┬─────────────┬─────┤
│  PostgreSQL │    Auth     │   Storage   │  Realtime   │ ... │
│   Database  │   (Users)   │   (Files)   │  (WebSocket)│     │
└─────────────┴─────────────┴─────────────┴─────────────┴─────┘
        │
        ▼
┌─────────────────────────────────────────────────────────────┐
│                    Tu Aplicación Node.js                     │
│              (Express + Prisma o Supabase Client)            │
└─────────────────────────────────────────────────────────────┘
```

### ¿Por qué Supabase + PostgreSQL?

| Característica | MongoDB Atlas | Supabase PostgreSQL |
|----------------|---------------|---------------------|
| Tipo de BD | NoSQL (documentos) | SQL (relacional) |
| Esquema | Flexible/schemaless | Estricto con tipos |
| Relaciones | Referencias manuales | Foreign keys nativas |
| Transacciones | Limitadas | ACID completas |
| Consultas | Aggregation Pipeline | SQL estándar |
| Escalabilidad | Horizontal | Vertical + Read replicas |
| Precio free tier | 512MB | 500MB + extras |

### Cuándo usar cada uno

**MongoDB es mejor para:**
- Datos con estructura variable
- Prototipado rápido
- Documentos anidados profundos
- Datos geoespaciales

**PostgreSQL es mejor para:**
- Datos con relaciones complejas
- Integridad referencial crítica
- Transacciones ACID
- Consultas SQL complejas
- Datos estructurados

## 2. PostgreSQL vs MongoDB

### Terminología

| MongoDB | PostgreSQL | Descripción |
|---------|------------|-------------|
| Database | Database | Contenedor principal |
| Collection | Table | Grupo de datos relacionados |
| Document | Row/Record | Un registro individual |
| Field | Column | Un atributo del registro |
| `_id` | `id` (PRIMARY KEY) | Identificador único |
| Embedded document | JSON/JSONB column | Datos anidados |
| Reference (`ObjectId`) | Foreign Key | Relación entre tablas |

### Comparativa de sintaxis

**Definir estructura:**

```javascript
// MongoDB/Mongoose
const userSchema = new Schema({
  name: { type: String, required: true },
  email: { type: String, unique: true },
  posts: [{ type: Schema.Types.ObjectId, ref: 'Post' }]
});
```

```prisma
// Prisma Schema
model User {
  id    Int     @id @default(autoincrement())
  name  String
  email String  @unique
  posts Post[]  // Relación automática
}
```

**Consultas:**

```javascript
// MongoDB/Mongoose
const user = await User.findById(id).populate('posts');
const users = await User.find({ age: { $gte: 18 } });
```

```javascript
// Prisma
const user = await prisma.user.findUnique({
  where: { id },
  include: { posts: true }
});
const users = await prisma.user.findMany({
  where: { age: { gte: 18 } }
});
```

## 3. Configuración de Supabase

### 3.1 Crear cuenta y proyecto

1. Ve a [supabase.com](https://supabase.com) y crea una cuenta
2. Crea un nuevo proyecto (elige región cercana, ej: eu-west)
3. Guarda la contraseña de la base de datos
4. Espera a que el proyecto se inicialice (~2 min)

### 3.2 Obtener credenciales

En el dashboard de Supabase, ve a **Settings → Database**:

```

│ Database Settings                                             │
├──────────────────────────────────────────────────────────────┤
│ Connection string (URI):                                      │
│ postgresql://postgres:[PASSWORD]@db.xxx.supabase.co:5432/postgres

└──────────────────────────────────────────────────────────────┘
```

### 3.3 Configurar variables de entorno

```env
# .env
DATABASE_URL="postgresql://postgres:[TU_PASSWORD]@db.xxxx.supabase.co:5432/postgres"

# Para conexiones con pooling (recomendado en producción)
DIRECT_URL="postgresql://postgres:[TU_PASSWORD]@db.xxxx.supabase.co:5432/postgres"
```

> **Importante**: En Supabase, usa el puerto `5432` para conexión directa o `6543` para connection pooling (Supavisor).

## 4. Prisma: El ORM Moderno

### 4.1 ¿Por qué Prisma?

**Prisma** es el ORM más popular para Node.js/TypeScript en 2025:

- **Type-safe**: Autocompletado y validación en tiempo de compilación
- **Migraciones automáticas**: Gestión de esquema integrada
- **Prisma Studio**: GUI para explorar datos
- **Rendimiento**: Queries optimizadas
- **Multi-database**: PostgreSQL, MySQL, SQLite, MongoDB, SQL Server

### 4.2 Instalación

```bash
# Crear proyecto
mkdir mi-api-supabase && cd mi-api-supabase
npm init -y

# Instalar Prisma
npm install prisma@6 --save-dev
npm install @prisma/client@6

# Inicializar Prisma con PostgreSQL
npx prisma init --datasource-provider postgresql
```

Esto crea:
```
mi-api-supabase/
├── prisma/
│   └── schema.prisma    # Definición del esquema
├── .env                 # Variables de entorno
└── package.json
```

## 5. Modelado de Datos con Prisma

### 5.1 Tipos de datos básicos

```prisma
model Example {
  // Identificadores
  id        Int      @id @default(autoincrement())  // Auto-increment
  uuid      String   @id @default(uuid())           // UUID
  cuid      String   @id @default(cuid())           // CUID

  // Strings
  name      String                    // VARCHAR(191)
  bio       String?                   // Nullable
  content   String   @db.Text         // TEXT (largo)

  // Números
  age       Int
  price     Float
  amount    Decimal  @db.Decimal(10, 2)  // Precisión exacta

  // Booleanos y fechas
  active    Boolean  @default(true)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  // JSON (PostgreSQL)
  metadata  Json?

  // Enums
  role      Role     @default(USER)
}

enum Role {
  USER
  AUTHOR
  ADMIN
}
```

### 5.2 Modelo de ejemplo: Blog

```prisma
// prisma/schema.prisma

model User {
  id        Int      @id @default(autoincrement())
  email     String   @unique
  name      String
  password  String
  role      Role     @default(USER)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  // Relaciones
  posts     Post[]
  comments  Comment[]

  @@map("users")  // Nombre de tabla en PostgreSQL
}

model Post {
  id          Int       @id @default(autoincrement())
  title       String
  slug        String    @unique
  content     String    @db.Text
  excerpt     String?
  published   Boolean   @default(false)
  publishedAt DateTime?
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  // Relaciones
  author      User      @relation(fields: [authorId], references: [id], onDelete: Cascade)
  authorId    Int
  comments    Comment[]
  categories  Category[]

  @@index([authorId])
  @@index([slug])
  @@map("posts")
}

model Comment {
  id        Int      @id @default(autoincrement())
  content   String
  createdAt DateTime @default(now())

  // Relaciones
  post      Post     @relation(fields: [postId], references: [id], onDelete: Cascade)
  postId    Int
  author    User     @relation(fields: [authorId], references: [id], onDelete: Cascade)
  authorId  Int

  @@map("comments")
}

model Category {
  id    Int    @id @default(autoincrement())
  name  String @unique
  slug  String @unique
  posts Post[]

  @@map("categories")
}

enum Role {
  USER
  AUTHOR
  ADMIN
}
```

### 5.3 Atributos importantes

| Atributo | Descripción |
|----------|-------------|
| `@id` | Clave primaria |
| `@unique` | Valor único |
| `@default()` | Valor por defecto |
| `@updatedAt` | Auto-actualiza timestamp |
| `@relation` | Define relación foreign key |
| `@map` | Nombre en BD diferente al modelo |
| `@@index` | Índice para performance |
| `@@map` | Nombre de tabla en BD |
| `?` | Campo nullable (opcional) |

### 6.2 CREATE - Crear registros

```javascript
// Crear un usuario
const user = await prisma.user.create({
  data: {
    email: 'ana@example.com',
    name: 'Ana García',
    password: hashedPassword,
    role: 'AUTHOR'
  }
});

// Crear con relación (post + autor)
const post = await prisma.post.create({
  data: {
    title: 'Mi primer post',
    slug: 'mi-primer-post',
    content: 'Contenido del post...',
    author: {
      connect: { id: userId }  // Conectar a usuario existente
    }
  },
  include: {
    author: true  // Incluir datos del autor en respuesta
  }
});

// Crear múltiples registros
const categories = await prisma.category.createMany({
  data: [
    { name: 'JavaScript', slug: 'javascript' },
    { name: 'Node.js', slug: 'nodejs' },
    { name: 'PostgreSQL', slug: 'postgresql' }
  ],
  skipDuplicates: true
});
```

### 6.3 READ - Leer registros

```javascript
// Buscar por ID
const user = await prisma.user.findUnique({
  where: { id: 1 }
});

// Buscar por campo único
const user = await prisma.user.findUnique({
  where: { email: 'ana@example.com' }
});

// Buscar uno (lanza error si no existe)
const post = await prisma.post.findUniqueOrThrow({
  where: { slug: 'mi-post' }
});

// Buscar primero que coincida
const post = await prisma.post.findFirst({
  where: { published: true },
  orderBy: { createdAt: 'desc' }
});

// Buscar todos
const posts = await prisma.post.findMany();

// Con filtros
const posts = await prisma.post.findMany({
  where: {
    published: true,
    author: {
      role: 'AUTHOR'
    }
  },
  include: {
    author: {
      select: { id: true, name: true }  // Solo ciertos campos
    },
    _count: {
      select: { comments: true }  // Contar comentarios
    }
  },
  orderBy: { createdAt: 'desc' },
  take: 10,   // Límite
  skip: 0     // Offset (paginación)
});
```

### 6.4 Filtros avanzados

```javascript
// Operadores de comparación
const users = await prisma.user.findMany({
  where: {
    age: { gte: 18 },              // >= 18
    name: { contains: 'García' },   // LIKE '%García%'
    email: { endsWith: '@gmail.com' },
    role: { in: ['AUTHOR', 'ADMIN'] },
    createdAt: {
      gte: new Date('2024-01-01'),
      lt: new Date('2025-01-01')
    }
  }
});

// OR / AND / NOT
const posts = await prisma.post.findMany({
  where: {
    OR: [
      { title: { contains: 'JavaScript' } },
      { title: { contains: 'Node' } }
    ],
    AND: [
      { published: true }
    ],
    NOT: {
      authorId: 5
    }
  }
});

// Filtrar por relaciones
const usersWithPosts = await prisma.user.findMany({
  where: {
    posts: {
      some: { published: true }  // Al menos un post publicado
    }
  }
});
```

### 6.5 UPDATE - Actualizar registros

```javascript
// Actualizar uno
const user = await prisma.user.update({
  where: { id: 1 },
  data: {
    name: 'Ana García López',
    role: 'ADMIN'
  }
});

// Actualizar o crear (upsert)
const user = await prisma.user.upsert({
  where: { email: 'nuevo@example.com' },
  update: { name: 'Nombre Actualizado' },
  create: {
    email: 'nuevo@example.com',
    name: 'Usuario Nuevo',
    password: hashedPassword
  }
});

// Actualizar muchos
const result = await prisma.post.updateMany({
  where: { published: false },
  data: { published: true }
});
console.log(`${result.count} posts publicados`);

// Incrementar/decrementar
const post = await prisma.post.update({
  where: { id: 1 },
  data: {
    views: { increment: 1 }
  }
});
```

### 6.6 DELETE - Eliminar registros

```javascript
// Eliminar uno

  where: { id: 1 }

// Eliminar muchos

  where: {
```

## 7. Relaciones en PostgreSQL

### 7.1 Tipos de relaciones

**One-to-Many (1:N)**
```prisma
model User {
  id    Int    @id @default(autoincrement())
  posts Post[] // Un usuario tiene muchos posts
}

model Post {
  id       Int  @id @default(autoincrement())
  author   User @relation(fields: [authorId], references: [id])
  authorId Int  // Foreign key
}
```

**Many-to-Many (N:M)**
```prisma
model Post {
  id         Int        @id @default(autoincrement())
  categories Category[] // Relación implícita
}

model Category {
  id    Int    @id @default(autoincrement())
  posts Post[] // Prisma crea tabla intermedia automáticamente
}
```

**One-to-One (1:1)**
```prisma
model User {
  id      Int      @id @default(autoincrement())
  profile Profile?
}

model Profile {
  id     Int  @id @default(autoincrement())
  bio    String
  user   User @relation(fields: [userId], references: [id])
  userId Int  @unique // Unique = 1:1
}
```

### 7.2 Operaciones con relaciones

```javascript
// Crear post con categorías (many-to-many)
const post = await prisma.post.create({
  data: {
    title: 'Aprendiendo Prisma',
    slug: 'aprendiendo-prisma',
    content: '...',
    author: { connect: { id: 1 } },
    categories: {
      connect: [
        { id: 1 },
        { id: 2 }
      ]
      // O crear nuevas:
      // create: [{ name: 'Prisma', slug: 'prisma' }]
    }
  },
  include: {
    author: true,
    categories: true
  }
});

// Actualizar relaciones
const post = await prisma.post.update({
  where: { id: 1 },
  data: {
    categories: {
      disconnect: [{ id: 1 }],  // Quitar categoría
      connect: [{ id: 3 }]       // Añadir otra
    }
  }
});

// Reemplazar todas las relaciones
const post = await prisma.post.update({
  where: { id: 1 },
  data: {
    categories: {
      set: [{ id: 2 }, { id: 3 }]  // Solo estas categorías
    }
  }
});
```

### 7.3 Incluir relaciones en consultas

```javascript
// Include: trae todos los campos
const post = await prisma.post.findUnique({
  where: { id: 1 },
  include: {
    author: true,
    categories: true,
    comments: {
      include: {
        author: true
      },
      orderBy: { createdAt: 'desc' },
      take: 5
    }
  }
});

// Select: solo campos específicos
const post = await prisma.post.findUnique({
  where: { id: 1 },
  select: {
    id: true,
    title: true,
    author: {
      select: {
        id: true,
        name: true
      }
    },
    _count: {
      select: {
        comments: true,
        categories: true
      }
    }
  }
});
```

## 8. Migraciones y Prisma Studio

### 8.1 Flujo de trabajo con migraciones

```bash
# 1. Después de modificar schema.prisma, crear migración
npx prisma migrate dev --name init

# Esto hace:
# - Genera SQL de migración
# - Aplica cambios a la BD
# - Regenera el cliente Prisma
```

```
prisma/
├── migrations/
│   └── 20250107120000_init/
│       └── migration.sql    # SQL generado
├── schema.prisma
```

### 8.2 Comandos de migración

```bash
# Desarrollo: crear y aplicar migración
npx prisma migrate dev --name add_comments

# Ver estado de migraciones
npx prisma migrate status

# Producción: aplicar migraciones pendientes
npx prisma migrate deploy

# Resetear BD (¡borra todos los datos!)
npx prisma migrate reset

# Generar cliente sin migrar (sincronizar tipos)
npx prisma generate
```

### 9.3 Controlador de Posts

```javascript
// src/controllers/posts.controller.js
import prisma from '../config/prisma.js';

// GET /api/posts
export const getPosts = async (req, res, next) => {
  try {
    const { page = 1, limit = 10, category } = req.query;
    const skip = (page - 1) * limit;

    const where = {
      published: true,
      ...(category && {
        categories: {
          some: { slug: category }
        }
      })
    };

    const [posts, total] = await Promise.all([
      prisma.post.findMany({
        where,
        include: {
          author: {
            select: { id: true, name: true }
          },
          categories: {
            select: { id: true, name: true, slug: true }
          },
          _count: {
            select: { comments: true }
          }
        },
        orderBy: { publishedAt: 'desc' },
        skip: Number(skip),
        take: Number(limit)
      }),
      prisma.post.count({ where })
    ]);

    res.json({
      data: posts,
      pagination: {
        page: Number(page),
        limit: Number(limit),
        total,
        pages: Math.ceil(total / limit)
      }
    });
  } catch (error) {
    next(error);
  }
};

// GET /api/posts/:slug
export const getPost = async (req, res, next) => {
  try {
    const post = await prisma.post.findUnique({
      where: { slug: req.params.slug },
      include: {
        author: {
          select: { id: true, name: true }
        },
        categories: true,
        comments: {
          include: {
            author: {
              select: { id: true, name: true }
            }
          },
          orderBy: { createdAt: 'desc' }
        }
      }
    });

    if (!post) {
      return res.status(404).json({
        error: true,
        message: 'Post no encontrado'
      });
    }

    res.json({ data: post });
  } catch (error) {
    next(error);
  }
};

// POST /api/posts
export const createPost = async (req, res, next) => {
  try {
    const { title, content, excerpt, categories } = req.body;
    const slug = title
      .toLowerCase()
      .replace(/[^a-z0-9]+/g, '-')
      .replace(/(^-|-$)/g, '');

    const post = await prisma.post.create({
      data: {
        title,
        slug,
        content,
        excerpt,
        author: { connect: { id: req.user.id } },
        categories: categories?.length ? {
          connect: categories.map(id => ({ id }))
        } : undefined
      },
      include: {
        author: { select: { id: true, name: true } },
        categories: true
      }
    });

    res.status(201).json({ data: post });
  } catch (error) {
    // Manejar slug duplicado
    if (error.code === 'P2002') {
      return res.status(409).json({
        error: true,
        message: 'Ya existe un post con ese título'
      });
    }
    next(error);
  }
};

// PUT /api/posts/:id
export const updatePost = async (req, res, next) => {
  try {
    const { id } = req.params;
    const { title, content, excerpt, published, categories } = req.body;

    // Verificar propiedad
    const existing = await prisma.post.findUnique({
      where: { id: Number(id) },
      select: { authorId: true }
    });

    if (!existing) {
      return res.status(404).json({
        error: true,
        message: 'Post no encontrado'
      });
    }

    if (existing.authorId !== req.user.id && req.user.role !== 'ADMIN') {
      return res.status(403).json({
        error: true,
        message: 'No tienes permisos para editar este post'
      });
    }

    const post = await prisma.post.update({
      where: { id: Number(id) },
      data: {
        title,
        content,
        excerpt,
        published,
        publishedAt: published ? new Date() : null,
        categories: categories ? {
          set: categories.map(id => ({ id }))
        } : undefined
      },
      include: {
        author: { select: { id: true, name: true } },
        categories: true
      }
    });

    res.json({ data: post });
  } catch (error) {
    next(error);
  }
};

// DELETE /api/posts/:id
export const deletePost = async (req, res, next) => {
```

### 9.4 Manejo de errores de Prisma

```javascript
// src/middleware/error.middleware.js
import { Prisma } from '@prisma/client';

export const errorHandler = (err, req, res, next) => {
  console.error(err);

  // Errores de Prisma
  if (err instanceof Prisma.PrismaClientKnownRequestError) {
    switch (err.code) {
      case 'P2002':
        return res.status(409).json({
          error: true,
          message: `El campo ${err.meta?.target} ya existe`
        });
      case 'P2025':
        return res.status(404).json({
          error: true,
          message: 'Registro no encontrado'
        });
      case 'P2003':
        return res.status(400).json({
          error: true,
          message: 'Error de referencia: el registro relacionado no existe'
        });
      default:
        return res.status(400).json({
          error: true,
          message: 'Error de base de datos',
          code: err.code
        });
    }
  }

  if (err instanceof Prisma.PrismaClientValidationError) {
    return res.status(400).json({
      error: true,
      message: 'Datos inválidos'
    });
  }

  // Error genérico
  res.status(err.status || 500).json({
    error: true,
    message: err.message || 'Error interno del servidor'
  });
};
```

### 9.5 Cerrar conexión correctamente

```javascript
// src/app.js
import express from 'express';
import prisma from './config/prisma.js';
import { errorHandler } from './middleware/error.middleware.js';
import postsRoutes from './routes/posts.routes.js';

app.use(express.json());
app.use('/api/posts', postsRoutes);

  console.log(`Servidor en http://localhost:${PORT}`);
});

// Cerrar conexión de Prisma al terminar
process.on('SIGINT', async () => {
  await prisma.$disconnect();
  server.close();
  process.exit(0);
});

process.on('SIGTERM', async () => {
  await prisma.$disconnect();
  server.close();
  process.exit(0);
});

export default app;
```
